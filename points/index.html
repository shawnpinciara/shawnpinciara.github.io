POINTS

Introduction:

POINTS is a project that aims to empower psychomotor coordination by giving a simple structure to space and time in a fun, affordable, and effective way.

The project is based on Points Notation, a way of writing movement
invented by me and the professional juggler Lucas Zileri. Using this easily learnable notation you'll have both access to an infinite variety of moves and to a clear idea of how to
organize them depending on you or a patient's needs and wants.

Because we want Points to arrive to as many people as possible, we've made it highly adaptable. There's no need for special objects, screens, or spaces: all you need is a basic understanding of the notation, some cards (which you can even download and print by yourself) and a few household objects, like cups, fruits..

The problems to be solved:

What if there was an easier way to teach juggling (based on rhythm)?

What if the exercises for muscular rehab become intellectually interesting and fun to do?

Idea and context:

I’ve been interested in juggling as an artistic and pedagogic discipline since 2011. I’ve never done professional studies in that field (apart from roughly 6 years of circus school) and that’s why I asked a professional juggler to help my project.

The linear progression while learning juggling has usually the objective of arriving learning the 3 balls cascade, which is considered to be the basic pattern of juggling (and the one seen represented in medias using 3 objects)

This progression has the big problem of proposing exercises in which the error rate (so the times you drop a ball) is always very high compared to the satisfaction of learning the skill itself.

More on that, usually the learnt cascade seems chaotic because of the lack of music rhythm understanding from the learner.

Then the approach is to use a rhythmic based training without throwing balls, but moving them on a table. Juggling usually takes place in the air but on a 2d plane in front of the person, so if we rotate this plane it becomes a surface where objects can be put.

This whole structure is called POINTS.

How does it work

POINTS presents a three-step process that’s very similar to how we approach piano playing: Read, Do, Create. The difference is that instead of learning what keys to press when and how you’ll learn what objects to move when and how.

1) Read

When you learn piano, the first step is learning to read the symbols that are on the score. The same applies to Points. Codified sequences will be presented and you’ll have to decipher them: your goal will be to understand what movements these sequences are asking you to do.

Memory and problem-solving skills will be trained in this part.

2) Do

Once you know what to do, you have to do it! As when learning the piano, knowing what’s written on the paper is just the first part: you then have to get the instructions “on the body”, coordinating both hands to move the objects through space and at a set rhythm.

Coordination will be the main skill trained in this part.

3) Create

The final part consists of creating your own sequences just like a music composer would do when he writes a new piece. You can try mixing the sequences you learned in different ways or you can even start from scratch and write your own sequence. The possibilities are endless!

Creativity and self-expression will be the focus of this final part.

Implementation:

The notation is based on 3 entities:

  • Objects that you move
  • Points in space where you put the objects onto
  • Medium that you use to move objects onto points (hands, mouth, wind)

The basic building block is a rectangle, inside it you find the path for the object:

An object is taken form position A and put to position B.

the POINTS are A and B, one near the other on a table like this:

Counting with the metronome, the object is taken at 1 and put down at 3.

Left to the rectangle there is the medium, like the right hand.

By stacking more lines, it means that both hands are moving objects.

Two interesting forms:

  • A hasn’t got a rectangle so the left hand is tapped at time 1 onto that point
  • The SNAKE at time 3: the objects is being passed from the right to the left hand, taken from position B with the right hand but put down still in B with the other hand.

These lines are then put in a bigger box in which the starting points (so how many points and how many objects in which points are there) and other information (such as duration, tempo…) can be defined:

Note that the starting position has 3 points, but only 2 are used by objects.

Flash cards and exercises (such as fill the white spaces, create sequences with the same starting or ending shapes) can be created starting from this framework, as it is highly customizable.

The position of cars in a parking or dancers in a choreography can be noted, as it is highly scalable.

Complex behaviours can emerge from really simple rules and blocks, as it is highly modular.

Where the project could go next

In the pedagogic world (schools) this project could be used to learn rhythm and psychomotor activities.

In the occupational therapy world it could be used for rehabilitation through movement (as the POINTS can be swapped by muscular action and objects could be gym weights).

Graphic Design:

To accomodate the need of using the project in an occupational therapy environment, the azure color was chosen. Based on that feeling of cleanness, I drawn a simple logo with the first letter of the project “PO”.

The P and O are based on a circle, that represent two different things:

  • Balls (circles) are used to perform the exercises
  • Both the P and O are actually points as you would write them in the notation: the P is full so it has an object on top, while the O is an empty space

A website was built with html, and vanilla css using only blocks. It is a single page app where both the project and a course is presented.
